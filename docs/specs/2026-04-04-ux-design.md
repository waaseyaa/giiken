# Giiken — UX Design Spec

**Date:** 2026-04-04
**Status:** Approved for implementation planning
**Author:** Russell Jones / Claudia

---

## Surfaces

Three distinct user experiences, two codebases.

| Surface | Audience | Technology |
|---|---|---|
| **Discovery** | Members, Staff | Inertia.js (single app, role-adaptive) |
| **Management** | Staff | Inertia.js (progressive disclosure within discovery app) |
| **Knowledge Keeper** | Knowledge Keepers, Admins | Waaseyaa admin SPA (Nuxt 3 / Vue 3 — existing) |

The Knowledge Keeper surface requires no additional UI work — they use the Waaseyaa admin SPA as-is. All Giiken-specific UI effort goes into the Inertia app.

---

## Design Principle: Progressive Disclosure

The discovery surface is the product for everyone. Staff don't go to a different URL — they see the same search/Q&A interface with a **Management** nav item that unfolds their tools.

When Management is open, the top nav transitions to a full sidebar. "← Back to Discover" is always visible at the bottom of the sidebar. Roles are additive: a staff member is also a community member who searches.

### Community Branding

Each community configures: name, logo, and optional primary colour override. These are applied as CSS custom property overrides at the tenant level. The Giiken base palette provides the fallback. No per-community stylesheet — one property swap.

### i18n

- Ships bilingual: English (default) and French
- All UI strings are in translation files — no hardcoded copy
- Additional languages (Ojibwe, Cree, etc.) added via translation file as a professional services engagement — no code changes required
- Waaseyaa `i18n` package (Layer 0) handles resolution

## Discovery Surface

### Layout

```
┌─────────────────────────────────────────────────────┐
│ Nav: Logo + Community Name | Discover  Recent  ⚙ Management  [Avatar] │
├─────────────────────────────────────────────────────┤
│                                                     │
│  [Indigo gradient hero]                             │
│  Community Name Knowledge Base                      │
│  Search or ask anything                             │
│  ┌─────────────────────────────────┐ [Ask →]        │
│  │ Search or ask a question…       │                │
│  └─────────────────────────────────┘                │
│                                                     │
├─────────────────────────────────────────────────────┤
│ Browse: 📖 Cultural  🏛 Governance  🌿 Land  👥 Relationships  📅 Events │
├─────────────────────────────────────────────────────┤
│ [Knowledge item cards]                              │
└─────────────────────────────────────────────────────┘
```

### Search vs Q&A — one input, two modes

The input field handles both. Mode is determined by query shape:
- Short keyword query (< 6 words, no `?`) → search results
- Full question (has `?` or > 5 words forming a sentence) → Q&A answer + citations + related results

Users do not choose a mode. The hint text shows examples of both.

### Q&A Answer State

```
[Search bar stays visible — indigo gradient — user can ask follow-up]
────────────────────────────────────────────────────
◉ Answer — from your knowledge base

  [Answer text with inline citation superscripts [1][2]]

  Sources
  [1] [Title] — [excerpt] — [type tag]
  [2] [Title] — [excerpt] — [type tag]
────────────────────────────────────────────────────
Related knowledge
  [Standard knowledge item cards]
```

Key rules:
- Answer is clearly labelled "from your knowledge base" — not the internet
- When no relevant items found: "I could not find information about this in the community's knowledge base" — no hallucinated answer
- Citations are expandable cards, not just footnotes
- Search bar stays in view for follow-up questions

### Knowledge Item Card

- Colour-coded dot (knowledge type)
- Title, excerpt (2 lines max)
- Type tag + relative date
- Clicking card opens full item detail view

## Management Panel (Staff)

Triggered by the **⚙ Management** nav item. The top nav transitions to a full left sidebar.

### Sidebar Structure

```
[Logo + Community]
────────────────
MANAGEMENT
  📊 Dashboard
  📋 Reports        ← default landing
  👥 Users
  📥 Ingestion Queue
────────────────
SYSTEM
  ⚙️  Settings
  📦 Export
────────────────
← Back to Discover
```

### Reports Page

- **Generate section:** three report type cards (Governance Summary, Language Report, Land Brief)
- Clicking a card opens a date-range picker + knowledge type filter → generates and downloads Markdown
- **Recent reports table:** title, type, generated date, download button
- Reports download as Markdown (`.md`) — PDF export is out of scope for v1

### Ingestion Queue Page

- List of files currently being processed through the compilation pipeline
- Shows: filename, stage (Transcribe / Classify / Structure / Link / Embed), status (queued / running / complete / failed)
- Failed items show error with retry button

### Users Page

- List of community users with roles
- Add user, change role, deactivate
- Role options: Admin, Knowledge Keeper, Staff, Member

### Export Page

- "Export all community data" button
- Shows what will be included: KnowledgeItems, media, embeddings, users, config
- Async export — user gets download link when ready
- Community Sovereignty Guarantee is surfaced here with plain-language explanation

## Knowledge Keeper Surface

Uses the Waaseyaa admin SPA (Nuxt 3 / Vue 3) without modification. Giiken registers its entity types (Community, KnowledgeItem) with the Waaseyaa admin, which auto-generates:

- List and detail views for KnowledgeItems
- Field editing (including access tier, allowed_roles, allowed_users)
- Media library access for source files
- Pipeline status (via Waaseyaa Telescope)

No additional admin UI work required in v1.

## Responsive Behaviour

Web-only (per product spec). Responsive targets:

| Breakpoint | Behaviour |
|---|---|
| Desktop (≥ 1024px) | Full layout as designed |
| Tablet (768–1023px) | Browse strip wraps, cards stack to single column |
| Mobile (< 768px) | Nav collapses to hamburger, hero shrinks, search full-width |

Management sidebar collapses to a top bar with icon-only nav on tablet/mobile.
